# Dirtywave M8

[Website](https://dirtywave.com) | [Manual](https://dirtywave.com/pages/resources) | Not open source | Hardware tracker by Timothy Lamb (Trash80)

## What it is

A handheld 8-track sequencer/synthesizer with an 8-button control model and a tracker-first UI. Despite the small form factor and limited controls, it packs a complete production environment: 7 synthesis engines, 128 instruments, effects, mixing, song arrangement, and a deep modulation system.

The M8 is the primary UI inspiration for helm-audio. Its constraint-driven design routes everything through a character grid with 8 buttons. The result is an interface that is remarkably information-dense without being cluttered.

## History

Timothy Lamb is Trash80, a chiptune musician known for Game Boy music and hardware projects like the arduinoboy (a MIDI-to-Game Boy interface). The M8 began as a DIY tracker project and later became a commercial product. It follows the tracker tradition (ProTracker → FastTracker 2 → LSDJ → M8) but with a modern synth engine.

The M8 also supports remote host display over USB. This is relevant because it proves the UI can be decoupled from the hardware screen for capture and desktop workflow.

## Display and layout

The M8's display is a character-grid UI. In practice, each major view keeps consistent regions for title/context, central content, and a right-side status area (tempo, track monitor, transport/minimap cues):

```
┌──────────────────────────────────────────────┐
│ Waveform oscilloscope (top strip, rows 0-1)  │
├──────────────────────────────────┬────────────┤
│ Title + context (row 2)         │ T▸128      │
├──────────────────────────────────┤            │
│                                  │ 1: A-3     │
│ Content area                     │ 2: ---     │
│ (view-specific)                  │ 3: ---     │
│                                  │ ...        │
│                                  │ 8: ---     │
│                                  ├────────────┤
│                                  │ ▐████▌     │
│                                  ├────────────┤
│                                  │   P        │
│                                  │ SCPIT      │
│                                  │   V        │
└──────────────────────────────────┴────────────┘
```

Across key views, you repeatedly see: title/context top-left, tempo top-right, right-side track monitor, and the SCPIT minimap in the lower-right area.

## Navigation model

Navigation is spatial. The core tracker flow is laid out horizontally (SCPIT), with vertical sub-pages accessible via Shift+Up/Down:

```
                    ┌─ Scale View
                    ├─ Groove View
PROJECT ← SONG ─ CHAIN ─ PHRASE ─ INSTRUMENT ─ TABLE
                              │         │
                              │         ├─ Modulation View
                              │         └─ Instrument Pool
                              │
                              └─ [connects to Table/Groove/Scale above]

          MIXER ─── Effect Settings
            ├─── EQ Editor
            └─── Limiter & Mix Scope
```

The horizontal axis moves between structure levels: song arrangement -> composition -> sound design. The vertical axis adds detail at each level. This gives the 8-button interface access to many views without feeling lost.

## The three MVP views

### Song View (our Pattern view)

An 8-track grid where each cell holds a chain reference (hex). Tracks play vertically through their chain list, looping at the first empty slot.

```
SONG                                          ████ █
                                              ▐▐▐▐▐▐
   1  2  3  4  5  6  7   8        T▸128
00▸00▸FD▸FD▸FD▸FD▸06▸00▸FD
01 04 08 02 03 05 17 07 09        1:A-3
02 11 13 0A 12 FD 10 FD 14        2:
03 FE 15 0F 18 0B 0E 01 0C        3:
...
```

Key behaviors:
- 256 rows of chains, 8 tracks. Each track plays independently.
- Different chain lengths cause tracks to drift apart. Each track position advances independently.
- Grayed-out chains (containing no notes) shown for readability.
- Live Mode ([SHIFT]+[LEFT]) lets you cue/stop tracks independently with quantized switching.
- Common convention: chain "00" or "FE" used as silent placeholders to maintain alignment between tracks.

### Phrase View (our Sequence view)

A 16-step sequencer for a single track. Six columns: note, velocity, instrument, and three effect commands.

```
PHRASE 3A*

  N    V  I   FX1       FX2       FX3     T▸128
0 A-4  66 02  KIL01     ---00     ---00   1:
1 ---  -- --  ---00     ---00     ---00   2:
...
```

Key behaviors:
- Empty cells are meaningful: note-off and KIL are explicit events, and leaving instrument empty (`--`) allows note edits without retriggering the instrument.
- **Three FX columns**: each holds a 3-char command + 2-char hex parameter. Commands are context-aware: sequencer FX (ARP, CHA, DEL, GRV, HOP, KIL, PSL, TBL, TIC, etc.) work on all instrument types; instrument FX control synthesis parameters.
- **Asterisk on title** (`PHRASE 3A*`): the phrase is shared, so editing affects all references.
- Velocity range: 00 (silence) to 7F (maximum), default 64.
- [EDIT]+[UP/DOWN] on an FX column opens the Effect Command Help view for browsable command selection.

### FM Synth Instrument View (our Instrument/Voice view)

The M8 has 7 engine types: Wavsynth, Macrosynth, Sampler, FM Synth, Hypersynth, External, MIDI Out. The FM Synth is most relevant to us.

```
INST. 00

TYPE    FMSYNTH        LOAD SAVE   T▸128
NAME    --------------
TRANSP. ON    TBL. TIC 03  EQ --    1: ---
                                    2: ---
ALGO    00 A▸B▸C▸D                  3: ---
        A SIN  B SIN  C SIN  D SIN  4: ---
RATIO   01.00  01.00  01.00  01.00  5: ---
LEV/FB  80/00  80/00  80/00  80/00  6: ---
MOD     -----  -----  -----  -----  7: ---
        -----  -----  -----  -----  8: ---
MOD1    00         AMP 00          ▐████▌
MOD2    00         LIM 00CLIP
MOD3    00         PAN 80█████▌
MOD4    00         DRY C0            M
FILTER  00 OFF     MFX 00          SCPIT
CUTOFF  FF ███████ DEL 00            V
RES     00         REV 00
```

**Common header** (all engine types):
- TYPE selector (cycles through engines, resets synth params but preserves mixer)
- NAME (editable label)
- TRANSP ON/OFF (disable for percussion)
- TBL (table link) + TIC (table speed)
- EQ slot assignment

**FM Synth parameters:**
- ALGO: 1 of 12 operator routing configurations, displayed as `A▸B▸C▸D`
- 4 operators (A-D) shown side by side with: waveform shape, RATIO (frequency multiplier), LEV/FB (level + feedback), and 2 MOD routing slots each
- MOD1-4: four modulation sources freely assignable to operator destinations (LEV, RAT, PIT, FBK)
- Operators can be copied/pasted between slots

**Mixer column** (right side, all engine types):
- AMP (amplification), LIM (limiter with modes: CLIP, SIN, FOLD, WRAP, POST, POST:AD, POST:W1-W3), PAN (with visual bar), DRY, MFX, DEL, REV send levels
- Visual indicators: PAN and CUTOFF show graphical bars alongside hex values

**Multi-mode filter** (all engine types):
- Type: OFF, LPF, HPF, BPF, BS, LP>HP, ZDF LPF, ZDF HPF
- Cutoff (00-FF) and Resonance (00-FF)

## The modulation system

Accessed below the Instrument View. 4 modulation slots per instrument, each configurable as:
- **AHD** (Attack-Hold-Decay)
- **ADSR** (Attack-Decay-Sustain-Release)
- **Drum** (specialized percussive envelope)
- **LFO** (with waveform shapes: triangle, sin, ramp, square, random, drunk, etc.)
- **Trig** (trigger envelope)
- **Tracking** (note/velocity tracking)

Modulation sources (MOD1-4) route into operators via the MOD slots on the instrument view. The routing is 1→2→3→4→1 for chaining. For example, MOD1 can modulate MOD2 depth.

## Effects and mixing

Three global send effects are available: ModFX (chorus/phaser/flanger), Delay (stereo ping-pong), and Reverb. Each has 4-5 parameters. Effects can chain, because ModFX and Delay both have a Reverb Send parameter. Each effect also has its own input EQ.

The Mixer View shows 8 tracks with per-track volume, input source (analog/USB), EQ assignment, and a limiter with visual scope.

Per-instrument mixer: the right-side column (AMP, LIM, PAN, DRY, sends) acts as per-voice output processing before the global mixer.

## Table View

A 16-row mini-sequencer runs alongside each instrument. It is like a phrase, but for automation. Each row has note transpose, volume, and 3 FX columns. Tables loop (configurable with HOP command) and run at a speed set by TIC. This is the M8's version of per-instrument macros. For example, if you want a wobble bass, you can place an LFO-like pattern of filter cutoff values in a table.

Table TIC modes include: triggered (TIC00), ticks-per-row (TIC01-TICFB), and mapped modes (TICFC=octave, TICFD=velocity, TICFE=note, TICFF=200Hz).

## Groove View

Per-step tick counts for timing. Default is 6 ticks per step (24 PPQ / 4 sixteenth notes = 6). Two rows at "06, 06" is straight time. "07, 05" is swing. "04, 04, 04" is sixteenth-note triplets. Grooves are assignable per-track via the GRV effect command.

Key UX detail: editing one row automatically adjusts the paired row to maintain the total tick count. A tick-sum help message appears at the bottom of the screen.

## What to be inspired by

- **The character grid as interface**: The M8 shows that a dense character-grid UI can still be fast and readable when layout is consistent.
- **The consistent right-side panel**: Across the major tracker views, track activity, tempo, and page indicator cues stay in familiar positions. This gives strong spatial awareness.
- **Spatial navigation**: Left/right for peer views, up/down for detail levels. You build a mental map of the interface as a 2D space.
- **Per-instrument mixer column**: Every voice has its own output stage (amp, limiter, pan, sends) before hitting the global mixer. This maps directly to our voice worklet architecture where each voice handles volume/pan/sends internally.
- **Table View as per-instrument automation**: A mini-sequencer per voice is more flexible than fixed LFO shapes. Tables can create arbitrary patterns of parameter changes.
- **The effect command system**: 3-char command + 2-char parameter, context-aware (different commands for different engine types). Dense, fast to type, easy to read.
- **Live Mode**: Cueing and stopping individual tracks independently. A stretch goal for us but the concept is proven.
- **The `*` indicator on shared phrases**: This simple marker is essential. It tells you that editing here affects other references.

## What to leave behind

- **The 8-button constraint**: We have a full keyboard. The M8's navigation (SHIFT+LEFT/RIGHT for pages, EDIT+DIRECTION for values) is shaped by having only 8 buttons. We should use direct keyboard shortcuts instead.
- **Chain View as a separate page**: The M8 uses Song → Chain → Phrase as three levels. For MVP, we skip chains and let pattern view reference sequences directly.
- **The SCPIT minimap**: Our pages are Pattern/Sequence/Instrument, not Song/Chain/Phrase/Instrument/Table. We need our own navigation indicator, not the M8's letter strip.
- **Fixed 8 tracks**: The M8 is hardware-limited to 8. We chose 8 for simplicity but the architecture supports more.
- **The waveform oscilloscope strip**: Nice for audio monitoring but not essential for MVP. Add it when we have AnalyserNode integration.
- **Hardware-specific engine types**: The M8 has Wavsynth, Macrosynth, Sampler, and more. We currently have one engine type (FM), so we do not need a TYPE selector.
